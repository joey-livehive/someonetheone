# someonetheone 설문 구성

`/start` 페이지 진입 시점부터 완료까지의 전체 흐름과 각 문항·선택지를 정리한 문서.

소스: `app/start/page.tsx`

## 전체 흐름 (Phase)

```
intro (9문항 객관식) → picky (자유 입력) → phone (전화번호 입력)
  → bridge (안내 화면) → detail (7문항 객관식)
  → photo (사진 업로드) → message (자유 입력) → done
```

---

## 1. INTRO — 상대방에 대한 객관식 (9문항)

> 변수명: `INTRO_QUESTIONS`
> 한 문항씩 보여주고 선택하면 자동으로 다음 문항으로 넘어감.

### Q1. 어떤 사람이 끌려?
- 부제: 생각하지 말고, 바로 골라봐
- 선택지:
  - '외모'가 수려한 사람 → `appearance`
  - '분위기' 좋은 사람 → `vibe`
  - '성격'이 좋은 사람 → `personality`
  - 유능한 '능력'을 가진 사람 → `competence`

### Q2. 나이는 어느 정도?
- 부제: 상대방 선호 나이대
- 선택지:
  - 나보다 어린 사람 → `younger`
  - 동갑이 좋아 → `same_age`
  - 나보다 많은 사람 → `older`
  - 상관없어 → `any_age`

### Q3. 키는 어느 정도?
- 부제: (없음)
- 선택지:
  - 작은 편이 좋아 → `short`
  - 비슷하면 좋겠어 → `similar`
  - 큰 편이 좋아 → `tall`
  - 상관없어 → `any_height`

### Q4. 체형은 어때?
- 부제: (없음)
- 선택지:
  - 마른 편 → `slim`
  - 보통 → `average`
  - 근육질 → `muscular`
  - 상관없어 → `any_body`

### Q5. 연애할 때 뭐가 제일 중요해?
- 부제: 하나만 고른다면
- 선택지:
  - 매일 연락하는 거 → `daily_talk`
  - 같이 있을 때 편한 거 → `comfortable`
  - 서로 응원해주는 거 → `support`
  - 취미를 같이 하는 거 → `shared_hobby`

### Q6. 연락 스타일은?
- 부제: 상대방한테 바라는 거
- 선택지:
  - 수시로 연락했으면 → `frequent`
  - 아침저녁 한 번씩 → `twice_daily`
  - 하루에 한 번이면 충분해 → `once_daily`
  - 연락 빈도는 상관없어 → `any_frequency`

### Q7. 종교는 중요해?
- 부제: (없음)
- 선택지 (3개):
  - 같은 종교였으면 → `same_religion`
  - 종교 없는 사람이 좋아 → `no_religion`
  - 상관없어 → `any_religion`

### Q8. 이건 좀 아닌 것 같아
- 부제: 절대 안 되는 거 하나
- 선택지:
  - 많이 바쁜 사람 → `too_busy`
  - 취미 없는 사람 → `no_hobby`
  - 친구 자주 만나는 사람 → `too_social`
  - 흡연하는 사람 → `smoker`

### Q9. 첫 만남은 어떤 게 좋아?
- 부제: (없음)
- 선택지:
  - 카페에서 가볍게 → `cafe`
  - 밥 먹으면서 → `meal`
  - 산책하면서 → `walk`
  - 뭐든 좋아 → `anything`

---

## 2. PICKY — 까다로운 기준 (자유 입력, 선택)

- 헤드라인: "말하기 힘든 / 나만의 까다로운 기준이 / 있다면?"
- 부제: 상대방은 전혀 모를 거야. 편하게 써줘.
- 입력 필드: textarea, 최대 500자
- 버튼 라벨:
  - 입력 있음 → `다음`
  - 비어있음 → `딱히 없어`
- 저장 시 question 라벨: `까다로운 기준`

---

## 3. PHONE — 전화번호 입력

- 헤드라인: 거의 다 왔어!
- 부제: 전화번호만 남겨줘. 찾는 대로 알려줄게.
- 입력: `010-1234-5678` 자동 포맷, 숫자 10~11자리 검증
- 버튼 라벨: `내 사람 찾아줘`

---

## 4. BRIDGE — 안내 화면

- 헤드라인: 고마워!
- 본문: 너에 대해서도 / 좀 더 자세히 물어봐도 돼?
- 버튼 라벨: `좋아, 물어봐`

---

## 5. DETAIL — 본인에 대한 객관식 (7문항)

> 변수명: `DETAIL_QUESTIONS`

### D1. 나이대가 어떻게 돼?
- 부제: 대략적으로
- 선택지:
  - 20대 초반 → `20_early`
  - 20대 후반 → `20_late`
  - 30대 초반 → `30_early`
  - 30대 후반 이상 → `30_late_plus`

### D2. 성별이 어떻게 돼?
- 부제: (없음)
- 선택지 (2개):
  - 남자 → `male`
  - 여자 → `female`

### D3. 어디쯤 살아?
- 부제: 대략적인 지역
- 선택지:
  - 서울 → `seoul`
  - 경기/인천 → `gyeonggi`
  - 그 외 지역 → `other_region`
  - 해외 → `overseas`

### D4. 주말에 보통 뭐 해?
- 부제: 제일 자주 하는 거
- 선택지:
  - 집에서 쉬어 → `rest_home`
  - 밖에 돌아다녀 → `go_out`
  - 사람 만나 → `meet_people`
  - 운동해 → `exercise`

### D5. 술은 어때?
- 부제: (없음)
- 선택지:
  - 자주 마셔 → `often`
  - 가끔 → `sometimes`
  - 거의 안 마셔 → `rarely`
  - 아예 안 마셔 → `never`

### D6. 연애 스타일은?
- 부제: 나한테 가까운 거
- 선택지 (3개):
  - 매일 보고 싶은 타입 → `clingy`
  - 적당한 거리가 좋아 → `moderate`
  - 각자 시간도 중요해 → `independent`

### D7. 지금 연애 준비됐어?
- 부제: 솔직하게
- 선택지 (3개):
  - 바로 만나고 싶어 → `ready_now`
  - 천천히 알아가고 싶어 → `take_slow`
  - 좋은 사람 있으면 → `if_right`

---

## 6. PHOTO — 사진 업로드 (선택)

- 헤드라인: 마지막! 사진 한 장
- 본문: 이 사진은 다른 사람에게 절대 공개 안 해. / 누군가 너와 비슷한 스타일을 찾는다면, 너에게 말해주려고 해.
- 동작: 파일 선택 (image/*) → base64로 변환 후 미리보기
- 버튼 라벨:
  - 사진 없음 → `사진 올리기` / `나중에 할게`
  - 사진 있음 → `다른 사진 선택` / `완료`

---

## 7. MESSAGE — 하고 싶은 말 (자유 입력, 선택)

- 헤드라인: 마지막으로,
- 부제: 나한테 하고 싶은 말 있어?
- 입력 필드: textarea, 최대 2,000자, placeholder `편하게 써줘 (선택)`
- 버튼 라벨:
  - 입력 있음 → `보내기`
  - 비어있음 → `괜찮아, 넘어갈게`
- 저장 시 question 라벨: `나한테 하고 싶은 말`

---

## 8. DONE — 완료 화면

- 헤드라인: 완벽해!
- 본문: 네 사람을 찾는 대로 알려줄게.
- 버튼: `돌아가기` (`/`로 이동)

---

## 요약 통계

| 구분 | 항목 수 | 비고 |
|---|---|---|
| INTRO 객관식 | 9문항 | Q7만 3지선다, 나머지는 4지선다 |
| DETAIL 객관식 | 7문항 | D2(2지선다), D6/D7(3지선다), 나머지 4지선다 |
| 객관식 합계 | **16문항** | |
| 자유 입력 | 2개 | picky(500자), message(2000자) — 둘 다 선택 |
| 기타 입력 | 2개 | 전화번호(필수), 사진(선택) |

## API 저장 동작 (참고)

- 설문 시작 시 `POST /theone/survey/start` → `guest_uid` 발급
- 객관식 답변: `PATCH /theone/survey/{uid}/answer` (body: `{ question, answer }`)
  - `question`은 화면 타이틀에서 줄바꿈만 공백으로 치환한 문자열
- 까다로운 기준: 같은 endpoint, `question = "까다로운 기준"`
- 메시지: 같은 endpoint, `question = "나한테 하고 싶은 말"`
- 전화번호: `PATCH /theone/survey/{uid}/phone`
- 사진: `PATCH /theone/survey/{uid}/photo` (base64 데이터)
